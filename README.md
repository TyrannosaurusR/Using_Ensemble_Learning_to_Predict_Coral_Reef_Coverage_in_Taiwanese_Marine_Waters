# Using Ensemble Learning to Predict Coral Reef Coverage in Taiwanese Marine Waters
此專案改良了我高中時期做的小論文-利用機器學習預測台灣珊瑚礁覆蓋面積。改良的部分利用集成式學習並實際的使用指標性生物以了解這些生物對於珊瑚礁生存的影響。這份Readme會比較偏向個人心得和進程，想知道研究在幹嘛的去看大學改良內部的書面資料。

## 高中時期
高中時期與朋友一同參加了全國中小學小論文競賽，因為先前有分析過大氣資料的經驗，想要做一些有趣的挑戰，因此便選定了珊瑚礁為主題並從更生物學的領域-指標性生物下手。
### 前期準備
透過閱讀文獻獲得珊瑚礁與指標性生物的背景知識，以及受到台灣資訊協會的協助獲得了台灣25個珊瑚礁生長地點2008年至2022年的資料，再比較幾篇論文了解了不同的機器學習方法。
### 模型訓練
當時的ChatGPT橫空出世，我們也利用此工具節省了許多查資料和debug的時間，再前前後後的清洗資料、選擇特徵、比較模型後，最後選定了決策樹、隨機森林、梯度提升樹這三個模型。
### 模型結果
在十次交叉驗後梯度提升樹使用岩石覆蓋率（RC）、營養鹽指標性藻類覆蓋率（NIA）、泥沙覆蓋
率（SI）、細沙覆蓋率（SD）、碎石覆蓋率（RB）、其他底值生物覆蓋率（OT）、
海綿覆蓋率（SP）、新死珊瑚覆蓋率（RKC）這些特徵得到了最好的結果，0.92的R-square和0.0365的RMSE。

雖然最後我們獲得了優等的佳績，但值得注意的是最後的特徵全都是覆蓋率，這等於我們的模型並沒有抓到個特徵間的複雜關係，而是單純注意到了 $珊瑚礁覆蓋率 = 1 - 其他覆蓋率$ ，但幸好評審沒有注意到(或只是單純沒說破而已)。

## 大學改良
大學時期學了更深層的統計以及修習了許多統計分析，為了更了解神經科學也特定跑了很多趟生科院上課，並多次運用所學在這個的專案上，嘗試在只保留生物相關數據的情況下得到較好的結果。此次終於有些許成果，並趁機學一下GitHub，把他丟上來。

### 基本統計
基本統計用在這個專題可謂是一踏糊塗，基本上挑不出好的特徵，雙向淘汰法下去通常所有的指標性生物都被刪光光。用了更多的特徵工程和資料清洗，結果會變好，但最好的R-square通常不會超過0.20。

### 集成式學習
大二寒假突然閒的沒事做，突然想說來問一下Gemini我這個專案有甚麼下手點，他給了我三個關鍵資訊，TabNet, XGBoost, 集成式學習。其中XGBoost在我高中問ChatGPT時就有得到一些資訊了，而集成式學習對我而言更屬於可遠觀不可褻玩焉者，但TabNet屬實沒聽過了，查了一下才發現這是一個超酷的東西，並正好搭配Colab新增的AI輔助來撰寫程式碼。之後在查閱資料後也添加CatBoost和LightGBM這兩項東西，並使用最後的集成式學習。得到的結果為MAE: 0.1081, MSE: 0.0186, R2 : 0.5923，得到了不錯的結果

### 氣象資料
又一次在與AI的對談中得知，Colab可以串Open-meto的API，簡而言之就是我可以透過查找經緯度與日期的方式來找到某地某時的氣象資料。之前礙於氣象資料很難找到很準確的地點或時間而放棄將其放入資料中，而再添加了天氣資料後，模型又變得更好了。

### 時間序列
在上完某一堂的工程統計後，突發奇想地去問了教授與課程內容毫不相干的Time series資料的分布問題，但教授也是很認真的替我解惑並告訴我現在的主流模型。雖然我很早就有聽過類似的觀念，但由於台灣資訊協會給的每個站店的測量時間並不多，再加上對此不熟，故之前一直沒有朝這個方向想。現在加入了Lag-feature和Prophet後，模型的解釋力和誤差得到了大大的進步，R2甚至到達了0.75的好分數。

## 結語
這一份資料從高中到現在，只要某些課的期末專題一有機會我就會拿這個專案出來玩，但現在很不幸的是我終於玩完了。最後的結果令我覺得已經好到足矣，故之後應該沒有甚麼機會再開這個專題了，所以才在這裡把它保存下來。當然我要學習的東西還很多，未來還有著各式各樣不同領域的專案在等著我。
